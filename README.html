Retriever Polytope

An automated Python data retrieval and processing pipeline for downloading meteorological, ocean, and climate data via the Destination Earth Polytope API.

Key Components

  • ▹
    main.py: The primary orchestrator script that maps out settings, handles the loop, and triggers processing.
  • ▹
    config.yaml: The central schema mapping defining output schemas, variables, streams, and resolutions.
  • ▹
    desp-authentication.py: Interacts with the Destination Earth IAM to generate Polytope OAuth2 tokens.
  • ▹
    src/ directory: Source modules handling extraction logic (auth.py, data_loader.py, processor.py).

Prerequisites & Setup

Prerequisites

  • ✓ Python 3.x
  • ✓ Dependencies:
    pyyaml python-dotenv requests lxml conflator pydantic
  • ✓ Polytope CLI and API libraries (required by data_loader.py)

Environment Security (.env)

Place your Destination Earth credentials in a `.env` file at the root repository:

DESP_PASSWORD="your_actual_password_here"

Your username should be maintained inside the config.yaml file.

Config File Details (config.yaml)

DestinE Variables List

The `config.yaml` file drives the data extraction. You must set your `user` and define your target datasets, parameters, and time ranges.

Essential Settings

  • user: Your DESP username.
  • output_dir: Template for dynamic sub-folders (e.g. `./output/{experiment}/{param}`).
  • live_request: Set to `true` to authenticate via DESP IAM.

Dataset Requests

  • param: The Variable ID you intend to extract (e.g. 235055).
  • date: Formatting specifies the range:
    20180101/to/20241231 or single days 20180101.
  • levtype: "sfc", "pl", "o2d", or "o3d".

Confused about which "param" (variable) ID or "dataset" names to use?
Refer to the complete interactive reference table outlining the mappings: Destination Earth Variables Overview

Processing Block

If you wish to convert GRIB data to NetCDF dynamically, set `nc: true` in the processing section. Also, specify the `target_param_name` specifically for what the NetCDF variable should be titled.

Execute the Pipeline

Once your environment and YAML configurations are established, start the orchestration script directly. It will systematically crawl the requested dataset, authenticate, parse out the GRIB file, and execute conversions dynamically if requested.

python main.py
